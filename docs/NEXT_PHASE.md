# Sellog 다음 단계 기획 (Phase 3 / 후속 작업)

AI Phase 1·2와 랜딩 Phase A 완료 후, **코드로 바로 진행 가능한** 개선 작업을 정리한 문서입니다.

---

## 1. 목표

- **SEO·메타 품질**: AI로 생성한 메타 설명을 DB에 저장해 OG/검색에 실제 반영
- **UX**: AI·폼 결과를 토스트로 일관되게 표시
- (추후) 다크 모드, 글 목록 필터 등 선택 확장

---

## 2. 구현 항목

### 2.1 메타 설명 저장 (SEO 반영) ✅

| 항목 | 내용 |
|------|------|
| **DB** | `posts.meta_description` 컬럼 추가 (마이그레이션 007) |
| **글쓰기** | 선택 입력 "메타 설명 (SEO)" 필드, AI 요약 생성 시 여기에 반영 |
| **글 수정** | 동일 필드, 기존 값 로드 |
| **저장** | createPostAction / updatePostAction에서 meta_description 전달 |
| **노출** | 글 상세 메타·OG에서 `meta_description` 우선 사용, 없으면 본문 excerpt |

**효과**: AI "요약 생성" 결과를 저장하면 검색·SNS 공유 시 해당 문구가 그대로 사용됨.

---

### 2.2 토스트 알림 ✅

| 항목 | 내용 |
|------|------|
| **라이브러리** | `sonner` (가벼운 토스트, Next 앱에서 흔히 사용) |
| **레이아웃** | 루트 layout에 `<Toaster />` 배치 |
| **사용처** | AI 성공 시 "제목이 채워졌어요" 등, AI 에러 시 "API 키를 설정해 주세요" 등, 폼 제출 에러 |

**효과**: 인라인 에러만 쓰던 부분을 토스트로 통일해 피드백이 눈에 잘 들어옴.

---

### 2.3 추후 후보 → 일부 구현 완료

| 항목 | 설명 | 상태 |
|------|------|------|
| **다크 모드** | `prefers-color-scheme` 반영 (스크립트로 html.dark), DomainHeader·BottomNav·글 목록 등 dark: 적용 | ✅ |
| **글 목록 필터** | 전체 / 초안 / 발행됨 탭 (PostsListWithFilter) | ✅ |
| **메타 설명 글자 수** | N/200자 표시, 160자 초과 시 권장 안내 (글쓰기·글 수정) | ✅ |
