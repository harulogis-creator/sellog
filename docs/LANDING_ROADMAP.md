## 2. 실행 로드맵 (우선순위)

### Phase A — 즉시 구현 (코드만으로 완료)

1. **사회적 증거 섹션**  
   - 수치 3개: "블로그 오픈", "글 발행", "상품 등록" 등 (플레이스홀더 또는 DB 집계 후 연동).
2. **FAQ 섹션**  
   - 6~8개 Q&A (Sellog 기준: 어떻게 만드나요, 비용, 도메인, SEO, 결제, 코딩 필요 여부 등).
3. **푸터 확장**  
   - 제품(기능·시작하기·가격·연결 상태), 지원(문서·연결 상태·문의), 회사(소개·문의), 소셜(플레이스홀더), 언어(한국어 | English 예정).
4. **기능 카드에 아이콘**  
   - lucide-react 아이콘으로 4카드 + 추가 2~4카드(총 6~8개).
5. **올인원 그리드**  
   - Squarespace 스타일 8개 미니 카드(블로그, 상품, SEO, 도메인, 결제, 설정, 글쓰기, 주문).
6. **"Sellog로 만든 블로그" 섹션**  
   - 플레이스홀더 4~6 카드 또는 DB에서 공개 블로그 N개 노출.
7. **상단 배너**  
   - "무료로 시작 · 신용카드 불필요" (닫기 가능).
8. **모바일 햄버거 메뉴**  
   - 768px 미만에서 네비 + 로그인/시작하기 드로어 또는 풀 메뉴.
9. **지원 메시지**  
   - 푸터 또는 섹션에 "문서·연결 상태에서 도움받기" 등 한 줄 안내.
10. **가격 섹션**  
    - "무료로 시작, 유료 플랜은 추후" CTA 1줄 + 버튼.

### Phase B — 콘텐츠·데이터 연동

- FAQ 확장, 실제 블로그 수/글 수 집계 후 사회적 증거 수치 교체.
- 가격 페이지(요금제) 추가 시 링크 연결.
- 문서 사이트 또는 GitHub Wiki 링크 확정 후 푸터 반영.

### Phase C — 선택(추후)

- 히어로 캐러셀(다중 메시지).
- 메가메뉴(드롭다운).
- i18n(English) 실제 전환.
